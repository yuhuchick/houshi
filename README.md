# 后室：潮湿灯影（houshi-backrooms）

一个基于 **Three.js + 原生 JavaScript + Socket.IO** 的轻量 3D 生存探索游戏。  
支持单人游玩、房间制多人联机、关卡密码推进与本地多存档。

## 项目特点

- 第一人称 3D 后室探索，动态迷宫单元生成
- 三关流程化关卡（线索收集 -> 密码解锁 -> 进入下一关）
- 多类型实体 AI（不同追击/巡逻/触发逻辑）
- 资源与生存系统（电量、恐惧、噪音、黑暗惩罚）
- 道具系统（电池、急救包、镇静剂、荧光粉、柜子躲藏）
- 本地多存档（自动/手动/另存为/删除/旧档迁移）
- Socket.IO 联机（房间创建、加入、远端玩家状态同步）

## 技术栈

- 前端：`HTML + CSS + JavaScript`（单文件主逻辑在 `index.html`）
- 3D 渲染：`three@0.128.0`（CDN）
- 实时通信：`socket.io-client`（CDN）+ `socket.io`（Node）
- 服务端：`Node.js + express + socket.io`

## 目录结构

```text

```

## 快速开始

### 1) 安装依赖

```bash
npm install
```

### 2) 启动服务

```bash
npm start
```

默认监听：`http://localhost:3000`

> 前端通过同源访问 `index.html`，并连接当前服务进行联机。

## 游戏流程（简版）

1. 进入游戏后在关卡内寻找 3 条线索纸条  
2. 前往出口门交互，打开密码面板输入三位密码  
3. 正确后进入下一关，最终关卡完成后通关

## 核心系统说明

### 1. 生存资源

- **电量**：手电开启时持续消耗，低电会有提示音
- **恐惧**：黑暗、实体接近会增长，急救类道具可降低
- **噪音**：移动与碰撞会抬升，影响部分实体追踪

### 2. 道具与交互

- 电池：立即回复手电电量
- 急救包：降低恐惧值
- 镇静剂：降低恐惧并短时减缓恐惧增长（有移动副作用）
- 荧光粉：可放置地面路标，缓解迷路
- 柜子：短时躲藏
- 假出口：触发负面事件（恐惧/噪音上升）

### 3. 敌对实体

- Smiler（笑魇）
- Partygoer（派对客）
- Skin Stealer（伪装者）
- Light Moth（灯蛾）
- Listener（听声者）

不同实体会依据光照、距离、噪音等条件切换行为。

### 4. 联机机制

- 通过房间号创建/加入房间（自动标准化为 8 位以内）
- 服务端维护房间成员
- 同步数据：关卡、位置、朝向、手电状态、表情与动作
- 支持玩家加入/离开广播

### 5. 存档机制

- 使用 `localStorage` 存储多存档
- 支持自动保存与手动保存
- 支持读取、删除、另存为
- 内置旧版单存档键迁移逻辑

## 操作说明（默认）

- 移动：`W/A/S/D` 或方向键
- 手电：点击 HUD 手电按钮
- 背包：点击 HUD 背包按钮
- 设置：点击 HUD 设置按钮
- 交互：屏幕中心瞄准可交互物后点击
- 放置荧光粉：`G`（或背包内点击“使用”）
- 表情动作：面板按钮或数字快捷键（`5~9`）

> 实际可操作项以界面内提示为准。

## 服务端事件协议（Socket.IO）

- 客户端 -> 服务端
  - `room:join`
  - `player:update`
  - `room:leave`
- 服务端 -> 客户端
  - `room:joined`
  - `player:joined`
  - `player:left`
  - `player:state`

## 开发建议

- 当前主逻辑集中在 `index.html`，后续可按模块拆分：
  - `scene`（渲染与地图）
  - `gameplay`（生存、实体、交互）
  - `network`（联机）
  - `save`（存档）
  - `ui`（HUD/弹窗）
- 可加入 TypeScript、ESM 构建流程（Vite）提升可维护性。
